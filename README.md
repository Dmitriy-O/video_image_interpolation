# Адаптивна інтерполяція відео на основі model-based кластеризації

**Інтерактивна демонстрація курсового проєкту**  
з дисципліни «Класифікація та кластеризація відеоданих»

## Мета проєкту

Демонстрація застосування методів **model-based кластеризації** для автоматичного виявлення типів контенту у відеоданих, на яких класична інтерполяція кадрів працює недостатньо добре.

Проєкт обґрунтовує доцільність **контент-адаптивної інтерполяції** та пропонує легкий, інтерпретований підхід на основі класичних алгоритмів курсу.

## Вибрані алгоритми

- **Ward (Agglomerative Hierarchical Clustering)** — мінімізує приріст внутрішньокластерної дисперсії.
- **KModes** — категоріальна model-based кластеризація (аналог ідей COOLCAT) на дискретизованих ознаках.

## Можливості

- Коректна **temporal** оцінка якості інтерполяції (ground-truth PSNR на triplets *i−1, i, i+1*)
- Порівняння Ward і KModes (ARI, NMI, точна згода)
- Профілі кластерів та репрезентативні кадри
- Інтерактивний дизайнер адаптивних стратегій (політики per cluster, Δ PSNR)
- Автоматичні рекомендації стратегій за профілями ознак
- Контроль seed та субдискретизації довжини відео

## Вибір відео для демонстрації помітного ефекту

Ефект адаптації (Δ PSNR) сильно залежить від контенту. На «звичайних» відео (рівномірний рух, розмова, повільні панорами) лінійна інтерполяція часто близька до оптимальної за PSNR — дельти маленькі або нульові.

Щоб побачити помітний позитивний ефект (і переконливу демонстрацію):

- Шукайте (або знімайте) сцени з **чіткими фазами руху**: спокій → різкий старт/прискорення (або навпаки — рух → зупинка). Приклади: м’яч, що підстрибує, кинутий предмет, людина, яка починає бігти, автомобіль, що рушає, двері, що відчиняються.
- Добре працюють: короткі спортивні фрагменти, handheld з періодами «статично → швидко», змонтовані кліпи (статичний кадр + вибух дії).
- Уникайте для демонстрації: рівномірний рух, низький рух, дуже гладке high-fps відео.

Практичні налаштування в інтерфейсі:
- `Крок PSNR (sample_step)` = 1 (більше triplet’ів, надійніша статистика).
- `Макс. кадрів` — 300–600 (менше агресивна субдискретизація, щоб не «знищити» короткі фази).
- Після аналізу дивіться вкладку «Якість» — бажано, щоб «Середній PSNR (temporal)» помітно відрізнявся між кластерами.
- Натисніть «Рекомендувати», потім за потреби підкоригуйте таблицю політик і «Порівняти».

Синтетичний приклад з тестів (`tests/test_core_logic.py`, `make_moving_frames`) з ідеальними мітками slow/fast дає чіткий виграш при використанні `hold` на швидкій фазі. Його легко перетворити на реальний тестовый кліп.

## Запуск

```bash
python -m venv venv
venv\Scripts\activate          # Windows
# source venv/bin/activate     # Linux / macOS
pip install -r requirements.txt
python app.py
```

**Детальне керівництво користувача** (пояснення параметрів, як налаштовувати аналіз, вибір відео, інтерпретація результатів) знаходиться у файлі [docs/user_guide.md](docs/user_guide.md).

## Як передати проєкт (легкий архів)

**Не надсилайте папку `venv/`** — вона займає сотні МБ і залежить від вашого ПК.

1. Запустіть упаковку:
   ```bash
   python pack_for_share.py
   ```
2. Надішліть файл з папки `dist/` (зазвичай **~50–150 KB** — лише код).
3. Отримувач створює своє venv і ставить залежності (див. «Запуск» вище).

Що **не потрібно** в архіві: `venv`, `__pycache__`, `.vscode`, відеофайли, тимчасові JSON з експорту.

## Публікація на GitHub

### 1. Створіть репозиторій на сайті

1. Увійдіть на [github.com](https://github.com).
2. **New repository** (зелена кнопка або `+` → New repository).
3. **Repository name:** наприклад `video-interpolation-clustering`.
4. **Public** (або Private для курсової).
5. **Не** ставте галочки «Add README» / «Add .gitignore» — вони вже є локально.
6. Натисніть **Create repository**.

### 2. Зв’яжіть локальний проєкт і завантажте код

У терміналі в папці проєкту (замініть `ВАШ_ЛОГІН` і назву репозиторію):

```bash
git remote add origin https://github.com/ВАШ_ЛОГІН/video-interpolation-clustering.git
git push -u origin main
```

При першому `push` GitHub попросить увійти (браузер або Personal Access Token).

### 3. Оновлення після змін

```bash
git add .
git commit -m "Опис змін"
git push
```

Локально вже є перший коміт на гілці `main`; папки `venv/` і `dist/` у репозиторій **не потрапляють** (див. `.gitignore`).

## Наукова цінність

- Практичне застосування методів курсу до задачі відеообробки.
- Легка адаптивна система без глибокого навчання.

## Методологічні принципи

1. Ознаки кадру: колір, текстура (Лапласіан), проксі руху.
2. Два незалежні model-based розбиття для перехресної перевірки.
3. Оцінка інтерполяції лише на часовій структурі (не випадкові пари в кластері).
4. Прості інтерпретовані політики: `linear`, `hold`, `biased`.
