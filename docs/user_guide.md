# Керівництво користувача

**Адаптивна інтерполяція відео на основі model-based кластеризації**

Це повне керівництво по використанню інтерактивного демонстраційного застосунку.

---

## Як користуватися (основний потік)

1. **Завантажте відео**  
   У лівому полі виберіть відеофайл. Відео повинно містити щонайменше 12 кадрів.

2. **Налаштуйте параметри аналізу** (права панель)  
   Дивіться детальний опис параметрів нижче.

3. Натисніть кнопку **«Запустити аналіз»**.

4. **Перегляньте результати**:
   - Вкладка **«Кластеризація»** — розподіл кадрів за кластерами (Ward і KModes).
   - Вкладка **«Профілі»** — середні значення ознак по кластерах + репрезентативні кадри.
   - Вкладка **«Якість»** — temporal PSNR для звичайної лінійної інтерполяції по кластерах.
   - Вкладка **«Стратегії»** — інтерактивний вибір політик + порівняння результатів.

5. У вкладці **«Стратегії»**:
   - Натисніть **«Рекомендувати»** (автоматичний підбір на основі профілів і виміряної якості).
   - За потреби відредагуйте таблицю політик вручну.
   - Натисніть **«Порівняти»**, щоб побачити Δ PSNR та Δ SSIM.

## Параметри аналізу

### Кластерів (k)
- Діапазон: 2–6
- **Що означає**: Кількість груп (кластерів), на які алгоритм розіб'є кадри.
- **Рекомендації**:
  - 3–4 — найчастіше оптимально.
  - Більше значення дає детальніший поділ, але кластери можуть стати дуже маленькими і менш інтерпретованими.

### Біни KModes
- Діапазон: 3–8
- **Що означає**: Кількість категорій, на які дискретизуються ознаки для алгоритму KModes.
- **Рекомендації**:
  - 5 — хороший баланс для більшості відео.
  - Більше значення — точніша категоризація, але чутливіше до шуму.

### Ознаки руху
- **Що означає**: Чи включати в ознаки різницю між сусідніми кадрами (motion_mean / motion_std).
- **Рекомендації**: Залишайте увімкненим у більшості випадків. Це одна з найважливіших ознак для виявлення динамічного контенту.

### Seed
- **Що означає**: Фіксоване "зерно" випадковості для кластеризації (Ward детермінований, KModes — ні) та семплінгу triplet'ів.
- **Рекомендації**: Змінюйте значення, щоб подивитися альтернативні варіанти розбиття на тому самому відео.

### Макс. кадрів
- Діапазон: 60–600
- **Що означає**: Максимальна кількість кадрів, яку буде взято з оригінального відео (з рівномірною субдискретизацією).
- **Рекомендації**:
  - 300 — типове значення для швидкого аналізу.
  - 500–600 — для більш точного аналізу (повільніше завантаження та обробка).

### Крок PSNR
- Діапазон: 1–5
- **Що означає**: Крок при виборі temporal triplet'ів (i-1, i, i+1) для оцінки якості інтерполяції.
- **Рекомендації**:
  - **1** — максимальна точність оцінки (оцінюються майже всі можливі позиції).
  - 2–3 — хороший компроміс швидкість/точність (використовується за замовчуванням).
  - Вищі значення — швидше, але менш надійна статистика.

## Робота з вкладками

### Кластеризація
Показує timeline розподілу міток для двох незалежних кластеризацій (Ward і KModes) та метрики узгодженості (ARI, NMI, точна згода).

### Профілі
- Таблиця середніх значень ознак по кластерах.
- Галерея репрезентативних кадрів кожного кластера.
- Допомагає зрозуміти, **чому** кадри потрапили в той чи інший кластер (високий рух, текстура, яскравість тощо).

### Якість
Оцінка якості **звичайної лінійної інтерполяції** на часовій структурі (temporal triplets).  
Чим нижчий середній PSNR у кластері — тим складніше для цього типу контенту стандартна інтерполяція.

### Стратегії
Інтерактивний дизайнер адаптивних політик:
- Таблиця з поточними політиками по кластерах (редагована).
- Кнопка **«Рекомендувати»** — автоматичний підбір на основі профілів ознак та виміряної якості.
- Кнопка **«Порівняти»** — симуляція адаптивної стратегії з розрахунком Δ PSNR та Δ SSIM.
- Графік порівняння та таблиця деталізації.

### Методологія
Коротка таблиця основних етапів алгоритму.

## Вибір відео для кращої демонстрації

Для отримання помітного ефекту адаптації найкраще підходять відео з **чіткими фазами руху**:

- Об'єкти, що починають/зупиняють рух (м'яч, людина, транспорт).
- Сцени зі статичними та динамічними частинами.
- Короткі фрагменти зі зміною швидкості або напрямку руху.

На відео з рівномірним рухом ефект адаптації зазвичай невеликий (це очікувано).

---

## Інтерпретація результатів

- **Низький PSNR у кластері** → цей тип контенту важкий для лінійної інтерполяції.
- **Позитивний Δ** після застосування `conservative` / `biased` / `motion_aware` → адаптивна стратегія дала покращення саме на цьому типі контенту.
- **SSIM** використовується як додаткова метрика (краще відображає структурну подібність, ніж чистий PSNR).
- Політики застосовуються з temporal smoothing, щоб уникнути хаотичного перемикання між сусідніми кадрами.

---

## Покращені стратегії (реалізовані в застосунку)

- `linear` — класичний бленд (α = 0.5)
- `conservative` — консервативний бленд (α = 0.25)
- `biased` / `motion_aware` — alpha залежить від рівня руху в кластері (чим вищий рух — тим консервативніше)
- `hold` — повторити попередній кадр (тільки для найскладніших кластерів)
- `forward_biased` — зсув до наступного кадру
